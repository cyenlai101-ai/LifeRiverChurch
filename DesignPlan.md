# Design Plan - 生命河基督教會官網暨牧養管理系統

本文件整理 UI/UX 設計方向、資訊架構、核心頁面與技術規劃，作為 Wireframe 與開發的基礎。

## 1) 設計目標
- 多站點一體化：中心總站 + 三分站可切換且視覺一致、識別清楚。
- 會員中心為核心：登入後直接進入最常用的奉獻、報名、代禱入口。
- 牧養優先：關懷與代禱流程操作直覺、可追蹤。
- 隱私保護：敏感資訊預設遮罩、權限嚴格分層。

## 2) 資訊架構 (IA)
- 公開區
  - 中心首頁 / 分站首頁
  - 活動列表 / 活動詳情
  - 教會介紹 / 小組資訊
  - 登入 / 註冊
- 會員中心
  - 儀表板 (每日金句、簽到 QR、我的小組)
  - 我的報名 / 我的奉獻 / 我的代禱
  - 個人資料與隱私設定
- 牧養系統 (後台)
  - 關懷對象管理 (CRM)
  - 關懷紀錄 Timeline
  - 代禱管理 / 代禱牆
  - 活動管理與報名控管
  - 統計報表 / Audit Log

## 3) 核心頁面 UX 規劃
- 中心首頁
  - Hero Video + 快速 CTA (奉獻 / 報名 / 代禱)
  - 分站切換列 + 本週活動
- 分站首頁
  - 以分站色系為主視覺
  - 主日資訊 / 小組入口 / 當月活動
- 會員中心 Dashboard
  - 上方：每日金句 + 個人簽到 QR
  - 中間：快速入口卡片
  - 下方：近期活動 / 代禱回覆
- 活動報名流程
  - Stepper 流程
  - 代理報名 / 團體報名
  - 報名完成 QR Code + 行前提醒
- 關懷 CRM
  - 人物卡 + Timeline
  - 語音轉文字入口
  - 轉介流程入口
- 代禱牆
  - 沉浸式背景 + 卡片瀑布流
  - 即時阿們互動數字

## 4) 視覺風格建議
- 主視覺：現代化、溫暖、呼吸感
- 分站色系：
  - 中心：藍
  - 光復：綠
  - 第二：橘
  - 府中：紫
- 卡片式設計 + 柔和陰影
- 重要資訊明顯層級區分，避免資訊疲勞

## 5) 互動與動效
- 分站切換保留視覺延續感 (平滑過渡)
- 代禱牆即時互動微動畫
- 報名流程進度條 / 完成提示動畫

## 6) 技術與系統建議
- SSO：一組帳號跨站點
- 權限：RBAC + site_id 隔離
- 活動報名：候補 + 代理 + 團體
- CRM：時間軸、評分指標、語音轉文字
- 通知：LINE 優先，Email/SMS 為備援
- Audit Log：敏感操作必留痕

## 7) MVP 建議範圍
- 中心首頁 / 分站首頁
- 登入 / 會員中心 Dashboard
- 活動報名基本流程
- 代禱牆基本互動
- CRM 基本關懷紀錄

## 8) Wireframe 內容清單
- 中心首頁
  - Hero Video、主 CTA、分站切換列、近期活動卡、最新消息
- 分站首頁
  - 分站識別色、主日資訊、小組入口、當月活動
- 登入 / 註冊
  - SSO 入口、LINE/Google/Apple/手機 OTP
- 會員中心 Dashboard
  - 每日金句、簽到 QR、快速入口卡、近期動態
- 活動列表 / 詳情
  - 分類篩選、活動卡、名額/候補提示、早鳥狀態
- 活動報名流程 (Stepper)
  - 報名人員、動態表單、付款方式、完成頁 + QR
- 代禱牆
  - 代禱卡片、阿們互動、隱私等級顯示
- CRM 關懷對象列表
  - 搜尋篩選、狀態標籤、快速新增
- 關懷紀錄詳情
  - Timeline、評分指標、語音轉文字入口、轉介
- 後台管理 (共用)
  - 活動管理、代禱審核、報表與 Audit Log

## 9) 分階段開發與測試清單

### Phase 0：規劃與基礎
- 交付
  - 需求凍結版 PRD、角色權限矩陣
  - Wireframe + 設計系統（色票/元件/排版）
  - API 規格與資料表草稿
- 測試
  - 需求走查（角色權限、隱私遮罩）
  - Wireframe 走查（關鍵流程可用性）

### Phase 1：架構與登入
- 交付
  - SSO/多站點導覽
  - 會員註冊/登入（LINE/Google/Apple/手機 OTP）
  - RBAC 權限與資料隔離機制
- 測試
  - Login/Logout 測試
  - 角色切換後可見功能驗證
  - 未授權存取封鎖測試

### Phase 2：公開站點 + 會員中心
- 交付
  - 中心首頁、分站首頁
  - 會員中心 Dashboard（每日金句/簽到 QR/快速入口）
- 測試
  - 多站切換一致性測試
  - Dashboard 元件完整性與權限測試

### Phase 3：活動報名流程
- 交付
  - 活動列表/詳情
  - 報名 Stepper（一般報名/代理/團體）
  - 自定義表單與付款流程（可先 mock）
- 測試
  - 報名流程 E2E
  - 欄位驗證 / 付款狀態流程
  - 候補/名額滿處理

### Phase 4：代禱系統
- 交付
  - 代禱提報 / 代禱牆
  - 審核流程與隱私等級
  - 即時互動（阿們計數）
- 測試
  - 隱私權限與匿名顯示測試
  - 即時互動同步測試

### Phase 5：關懷 CRM
- 交付
  - 關懷對象管理 + Timeline
  - 生命成長指標、提醒機制
  - 轉介流程
- 測試
  - Timeline 記錄寫入/查詢
  - 提醒規則與時間條件測試
  - 角色資料可視性測試

### Phase 6：後台管理與統計
- 交付
  - 活動管理/代禱管理/報表
  - 跨站統計 Dashboard
  - Audit Log
- 測試
  - 報表數據一致性
  - Audit Log 完整性驗證
  - 跨站資料隔離測試

### Phase 7：整體驗收與上線
- 交付
  - E2E 測試套件
  - 效能/安全測試報告
  - 使用者操作說明文件
- 測試
  - 大量報名壓測
  - 故障情境（離線簽到、金流失敗）
  - 角色權限最終驗證
